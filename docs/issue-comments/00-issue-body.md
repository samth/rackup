## Windows Support for rackup

PLAN.md states: "V1 is Linux-host only, but code is structured for future macOS/Windows support." This issue tracks research and planning for adding Windows support.

### Current State

rackup v1 is Linux-only. Several components are deeply Unix-specific:
- **Bootstrap installer**: POSIX shell script (`install.sh`)
- **Shim dispatcher**: Embedded Bash script generated by `shims.rkt`
- **Shell integration**: Bash/Zsh only (`shell.rkt`)
- **Symlinks**: Used extensively for shims, toolchain bin dirs, and runtime links
- **Installer execution**: Runs Racket's `.sh` installers with `--create-dir --in-place --dest`

### Research Summary

Comprehensive research has been conducted on how 9 toolchain managers handle Windows support:
- **rustup** (Rust), **GHCup** (Haskell), **Volta** (JavaScript), **juliaup** (Julia)
- **nvm-windows** (Node.js), **pyenv-win** (Python), **mise** (polyglot), **Scoop**, **SDKMAN** (JVM)

Full research document: `docs/windows-support-research.md` on branch `claude/research-windows-support-ZAvnA`

### Key Decisions Needed

1. **Shim strategy**: Native `.exe` dispatcher (proven by rustup/Volta/GHCup/Scoop) vs script shims (fragile)
2. **Bootstrap method**: PowerShell script (like GHCup) vs standalone `.exe` (like rustup)
3. **Racket installer format**: Use `.tgz` archives (extractable) since Windows `.exe` installers lack `--in-place` CLI flags
4. **Default install location**: Short path needed (e.g., `%LOCALAPPDATA%\rackup`) to avoid MAX_PATH issues
